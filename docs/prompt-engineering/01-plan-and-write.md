## 초록

- 모델 재훈련 없이 정밀한 길이 제어를 가능하게 하는 프롬프트 엔지니어링 방법론 제시
- 프롬프트 내에 계획 수립과 단어 수 세기 메커니즘을 구현하여 모델이 지정된 길이 제약을 신중하게 추적하고 준수하도록 유도함
- 문서 요약 작업에 적용될 때  크게 향상시키는 것으로 나타남
    - 특히 짧은 길이에서 중간 길이 제약에서 효과적이였음
    - 다양한 모델 아키텍처마다 효과는 다르지만 최대 37.6% 개선을 보였음
- 길이 제어가 필요한 프로그램에 즉시 배포 가능한 솔루션을 제공함

## 1. 서론

- 응답 길이를 정밀하게 제어하는 능력이 중요한 요구 사항으로 부상하고 있음
- 다양한 사용 사례에서 다양한 응답 길이가 요구됨
    - 음성 인터페이스 : 간결한 답변 필요
    - 연구 요약 : 포괄적인 세부 사항 필요
    - 모바일 애플리케이션 : 화면 공간 제약 → 길이 제약
    - 문서화 시스템 : 다양한 수준의 정보 밀도 필요
- 하지만 LLM이 발전해도 명시적인 출력 길이 제약을 위반하고 있음
- 해결방안들이 많은데, 모델 재훈련, 복잡한 파인튜닝, 등 비용 및 시간이 많이듬
- 이 연구에서는 프롬프트 엔지니어링만으로도 이러한 문제들을 해결하려함

## 2. 관련 연구

- **기존 접근 방식:** LLM의 출력 길이 제어를 위해 모델 파라미터 미세조정(Fine-tuning)에 집중
    - **T-REG (Zhou et al.):** 토큰 수준 보상 정규화 도입
    - **RLHF 개선 (Park et al.):** 길이 악용 방지를 위한 정규화 항 추가
    - **인과 추론 (Dubois et al.):** 장황함 편향 제거 프레임워크 제안
- **한계점:** 재훈련 비용 과다, 복잡한 파이프라인, 블랙박스 모델 적용 불가 등 실용성 부족
- **본 연구의 차별점:** 재훈련 없이 **프롬프트 엔지니어링**만으로 문제 해결 시도

⇒  결과는 좋을 수 있으나 비용이 많이드는 재훈련, 파인튜닝, 블랙박스 LLM 포함하는 등 복잡하고 비실용적임

## 방법론

### 3.1 개요

- **핵심 개념:** 모델이 콘텐츠 작성 시 길이 인식(Length Awareness)을 갖도록 유도
- **구조:** 계획(Planning) → 검증(Verification)의 2단계 접근

### 3.2 프레임워크

길이 제어 생성 문제를 다음과 같이 공식적으로 정의합니다: 문서 D, 목표 단어 수 t, LLM M이 주어졌을 때, D에 대한 의미적 관련성을 최대화하면서 목표 길이를 엄격히 준수하는 요약 S를 찾음

**S = arg max Quality(s, D)  subject to |s| = t**  (수식 1)

(|s|는 요약 s의 단어 수를 나타내고, Quality(s, D)는 요약과 원본 문서 간의 의미적 유사성 및 관련성을 측정)

- **단계 1 (계획):** 콘텐츠를 생성하며 각 단어에 번호를 매겨 명시적으로 카운팅 (메타인지 활용)
- **단계 2 (검증):** 카운팅 된 내용을 바탕으로 정확한 길이의 일관된 문단으로 재구성

### 3.3 정당성

- **명시적 모니터링:** 추상적인 길이 제약을 구체적인 절차적 작업(단어 세기)으로 변환
- **2단계 구조:** '길이 맞추기'와 '글 다듬기'로 인지 부하 분산
- **기존 기능 활용:** 모델의 기본적인 산술 및 지시 이행 능력만 활용 (별도 학습 불필요)

## 4. 실험

### 4.1 실험 설정

#### 4.1.1 사용 모델 :

- Claude 3 Haiku
- Claude 3.5 Haiku
- Claude 3.5 Sonnet
- Claude 3.7 Sonnet
- Mistral Large
- Meta의 Llama 3.1 70B

#### 4.1.2 작업 목표 길이

- PDF 문서를 주고, 다양한 길이의 요약을 만들도록 지시함
- 20, 50, 100, 200, 500, 1000, 2000, 5000단어

#### 4.1.3 평가 메트릭

- MAPD 사용 (생성된 단어 수와 목표 단어 수 사이의 백분율 오차)
- **MAPD = |생성된_단어 - 목표_단어| / 목표_단어**

### 4.2 구현 세부 사항

#### 4.2.2 프롬프트 변형

- 4개의 프롬프트 전략 설계 및 평가
- **(1) 바닐라 V1:** 목표 단어 수로 요약하라는 간단한 지시
    
```text

Summarize this document into exactly {target_words}words.

---

이 문서를 정확히 {목표_단어} 단어로 요약하세요

```
    
- **(2) 바닐라 V2:** 다른 표현을 사용한 표준 프롬프트의 변형
    
```text

Transform this document into exactly {target_words}words.

---

이 문서를 정확히 {목표_단어} 단어로 변환하세요.

```
    
- **(3) 사고 V1:** 명시적 단어 수 세기를 포함한 Plan-and-Write 접근 방식.
    
```text

YOUR ONLY TASK: Summarize this document in EXACTLY {target_words} WORDS.
I will only accept summaries with EXACTLY {target_words} words - count carefully.
STEP 1: Count each word as you write:
<thinking>
1 First
2 word
...
{target_words} lastword
</thinking>
STEP 2: Rewrite those SAME {target_words} WORDS
in paragraph form:
<final_answer>
First word...
[EXACTLY {target_words} WORDS TOTAL]
</final_answer>

---

당신의 유일한 과제: 이 문서를 정확히 {목표_단어} 단어로 요약하세요.
정확히 {목표_단어} 단어인 요약만 수락합니다 - 신중하게 세세요.

단계 1: 작성하면서 각 단어를 세세요:
<thinking>
1 첫번째 단어
2 두번째 단어
...
{목표_단어} 마지막단어
</thinking>

단계 2: 동일한 {목표_단어} 단어를 문단 형식으로 다시 작성하세요:
<final_answer>
첫번째 단어...
[총 정확히 {목표_단어} 단어]
</final_answer>

```
    
- **(4) 사고 V2:** 구조화된 계획이 있는 과학적 프레이밍의 접근 방식.
    
```text

TASK: Transform this document to EXACTLY {target_words} words while maximizing information
preservation.
SCIENTIFIC METHODOLOGY:
(a) First, identify the core information hierarchy
and key points
(b) Then perform controlled expansion to EXACTLY {target_words} words by:
• Preserving primary information structures
• Including supporting details proportionally
• Maintaining relative emphasis on topics from
original document
• Adding clarifying context where needed to
reach target length
EXECUTION:
<thinking>

- First outline core information structure
- Draft initial version
- Systematically expand by adding supporting
examples
- Count words meticulously: 1, 2, 3...
until reaching {target_words}
</thinking>
Final {target_words}-word document:

---

과제: 정보 보존을 최대화하면서 이 문서를 정확히 {목표_단어} 단어로 변환하세요.

과학적 방법론:
(a) 먼저, 핵심 정보 계층 구조와 주요 포인트를 식별하세요
(b) 그런 다음 정확히 {목표_단어} 단어로 제어된 확장을 수행하세요:
• 주요 정보 구조 보존
• 지원 세부 사항을 비례적으로 포함
• 원본 문서의 주제에 대한 상대적 강조 유지
• 목표 길이에 도달하기 위해 필요한 곳에 명확한 맥락 추가

실행:
<thinking>

- 먼저 핵심 정보 구조 개요 작성
- 초기 버전 초안 작성
- 지원 예시를 추가하여 체계적으로 확장
- 단어를 꼼꼼히 세기: 1, 2, 3...
{목표_단어}에 도달할 때까지
</thinking>

최종 {목표_단어} 단어 문서:

```
    

#### 4.2.3 생성 프로토콜

- 6개 모델 x 8개의 길이 x 5번시도 x 4개의 프롬프트 변형 = 960개 시행

#### 4.2.4 단어 수 세기 방법론

- NLTK의 word_tokenize 함수를 사용해서 구두점을 제외
- "사고" 프롬프트 변형의 경우 단어 수 정확도를 평가하기 위해 지정된 태그 사이에서 최종 답변 텍스트만 추출

- **Plan-and-Write 우수성:** 6개 모델 중 4개에서 표준 프롬프트 대비 성능 우위
    - 특히 **Claude 3.7 Sonnet**에서 '사고 V1' 적용 시 기존 대비 **37.6% 개선** (MAPD 0.088)
- **모델별 특이점:** **Llama 3.1 70B**는 기본 프롬프트(바닐라 V2)에서도 이미 높은 제어력(MAPD 0.027)을 보여 개선 효과 미미
- **길이 스케일링:** 목표 길이가 길수록(500단어 이상) 준수율이 높아지는 경향

### 5.1 LLM-as-a-Judge를 사용한 품질 평가

- LLM-as-a-Judge(LLMaaJ)을 사용해서 평가를 진행함
    - **정확성(Correctness):** 소스 문서에 상대적인 요약에 제시된 정보의 사실적 정확성 측정
    - **완전성(Completeness):** 요약이 길이 목표에 비례하여 원본 문서의 모든 필수 정보를 포착하는지 평가
    - **충실성(Faithfulness):** 요약에 원본에 없는 사실이나 주장을 도입하지 않고 소스 문서와 일치하는 정보가 포함되어 있는지 평가
    - **관련성(Relevance):** 요약이 핵심 주제를 강조하면서 부차적인 세부 사항을 피하며 문서의 핵심 메시지에 중요한 정보에 얼마나 잘 집중하는지 측정

### **5.2 오픈 웨이트 모델 평가**

- Qwen 2.5 7b 모델로 추가 실험
- 우리의 발견에 따르면 사고 V2 접근 방식이 가장 낮은 전체 MAPD(0.280 ± 0.636)를 달성하여 바닐라 V2(0.281 ± 0.169)를 약간 능가

### 5.3 트레이트오프 분석

- 사고 프롬프트는 바닐라에 비해 약간 더 많은 토큰을 사용(1.02x)
- 그러나 추론 시간이 상당히 길어짐(1.57x)

## 6. 한계점

- **모델 의존성:** 이미 길이 제어 성능이 뛰어난 모델(예: Llama 3.1)에서는 이득이 제한적
- **초장문 제약:** 5,000단어 이상의 긴 목표에서는 정밀한 계산 능력 저하로 충실도 감소
- **비용 증가:** 2단계 생성 과정(계획+작성)으로 인한 추론 비용 및 시간 오버헤드 발생
- **평가 편향:** LLM-as-a-Judge 평가 방식 자체의 잠재적 편향 존재

## 7. 결론

- **핵심 기여:** 재훈련 없는 프롬프트 엔지니어링(Plan-and-Write)으로 길이 제어 정밀도 획기적 개선 입증
- **실용성:** 음성 인터페이스, 모바일 앱 등 길이 제약이 엄격한 환경에 즉시 적용 가능한 솔루션 제공
- **향후 과제:**
    - 프롬프트와 경량 추론 수정을 결합한 하이브리드 접근
    - 길이 외 다른 제약 조건(스타일 등)으로의 확장성 연구

- 요약
    
    **문제**: LLM이 "정확히 50단어로 요약해줘" 같은 길이 제약을 잘 지키지 못함
    
    **해결책**: 모델 재훈련 없이 프롬프트만으로 해결하는 2단계 방식
    
    1. **계획 단계**: 글을 쓰면서 각 단어에 번호를 매겨 카운팅
    2. **검증 단계**: 카운팅한 내용을 깔끔한 문단으로 재구성
    
    **V1 스타일 프롬프트 :** 
    
    기본형 :

    ```text
    당신의 유일한 과제: 이 문서를 정확히 {목표_단어}단어로 요약하세요.
    정확히 {목표_단어}단어인 요약만 수락합니다 - 신중하게 세세요.

    단계 1: 작성하면서 각 단어를 세세요:
    <thinking>
    1 이
    2 연구의
    3 LLM의
    ...
    {목표_단어} 마지막단어
    </thinking>

    단계 2: 동일한 {목표_단어}단어를 문단 형식으로 다시 작성하세요:
    <final_answer>
    [총 정확히 {목표_단어}단어]
    </final_answer>
    ```
