## LLM의 개인 글쓰기 스타일 모방 능력 평가 연구

### 연구 배경 및 동기

- LLM은 방대한 웹 데이터를 학습했기 때문에 일반적이고 평균적인 스타일로 글을 쓰는 경향이 있음 → 개인의 고유한 스타일과 진정성 있는 글쓰기 능력이 떨어짐
- 이 문제를 해결하기 위해 few-shot 프롬프팅을 활용하여 개인의 스타일을 모방할 수 있을지 평가

### **평가 접근법**

- 본 논문에서는 소수의 사용자 작성 샘플로부터 맥락 내 학습(in-context learning)을 통해 개인 글쓰기 스타일 모방 능력에 대한 포괄적 평가를 제시
    - **저자 귀속(Authorship Attribution, AA)**: 후보 저자 집합에서 주어진 텍스트의 저자를 식별
    - **저자 검증(Authorship Verification, AV)**: 두 텍스트가 같은 저자의 것인지 판단
    - **스타일 매칭**: 생성된 텍스트가 저자 고유의 스타일 패턴과 얼마나 일치하는지 측정
    - **AI 탐지**: 생성된 텍스트가 AI가 아닌 사람이 쓴 것으로 탐지되는지 확인

### **평가 규모**

- 뉴스, 이메일, 포럼, 블로그 등 4가지 영역
- 400명 이상의 실제 저자
- 모델당 40,000개 이상의 생성물

**결과 미리보기** : 
아직까지는 **뉴스와 이메일 같은 구조화되고 공식적인 형식에서는 어느 정도 사용자 스타일에 근접**할 수 있지만, **블로그와 포럼의 미묘한 차이가 있는 비공식적인 글쓰기에서는 상당한 어려움**을 겪는다는 것을 발견

### 방법론

> LLM이 **명시적인 스타일 지시 없이** 소수의 예시만으로 사용자의 **암묵적 글쓰기 스타일**을 모방할 수 있는지 평가
> 

현실적인 사용 시나리오를 반영하여:

- 사용자가 소수의 글쓰기 샘플과 콘텐츠 요약만 제공
- 전형적인 few-shot 프롬프팅 방식 사용
- 스타일을 상세히 설명하지 않음
- **RAG**(Retrieval-Augmented Generation, 검색 증강 생성)나 **Trial-Error-Explain** 같은 복잡한 고급 기법은 사용하지 않음
    
    > 참고: RAG는 외부 지식 베이스를 검색하여 생성에 활용하는 기법이고, Trial-Error-Explain은 여러 번의 시행착오를 통해 점진적으로 개선하는 방법입니다. 이러한 기법들은 효과적이지만 실시간 사용에는 비실용적
    > 
    

#### (A) 생성 과정

1. 각 저자의 글쓰기 샘플을 **훈련 세트**와 **테스트 세트**로 분할
2. 훈련 샘플에서 few-shot 예시를 선택
3. LLM이 테스트 샘플의 요약을 기반으로 새로운 텍스트를 생성하도록 안내

#### (B) 평가 방법 (4가지 지표)

1. **AA (저자 귀속, Authorship Attribution)**:
    - 모델이 LLM 생성 샘플을 대상 저자가 작성한 것으로 정확히 예측하는가?
    - 여러 후보 중에서 올바른 저자를 식별하는 과제
2. **AV (저자 검증, Authorship Verification)**:
    - 모델이 LLM 생성 샘플과 인간 작성 샘플을 같은 저자의 것으로 판단하는가?
    - 두 텍스트의 저자 일치 여부를 판별하는 이진 분류 과제
3. **스타일 분석 (Style Analysis)**:
    - LLM 생성 샘플과 인간 작성 샘플의 스타일 분포가 얼마나 유사한가?
    - 언어적 특징의 통계적 분포를 비교
4. **AI 생성 탐지 (AI Detection)**:
    - LLM 생성 샘플이 비AI 텍스트(즉, 사람이 쓴 글)로 탐지될 수 있는가?
    - 사람다움(human-likeness)의 지표

#### 데이터셋

4가지 다양한 장르의 데이터셋을 사용, 총 400명의 저자, 다양한 글쓰기 스타일을 포괄:

| 데이터셋 | 장르 | 저자 수 | 샘플 수 | 평균 길이 |
| --- | --- | --- | --- | --- |
| **Enron** | 이메일 | 150명 | 3,884개 | 309단어 |
| **Blog** | 블로그 | 100명 | 25,224개 | 319단어 |
| **CCAT50** | 뉴스 기사 | 50명 | 2,500개 | 584단어 |
| **Reddit** | 온라인 포럼 | 100명 | 8,451개 | 333단어 |

### 평가 프레임워크의 타당성 검증

| 지표 | 성능 | 의미 |
| --- | --- | --- |
| **AV 정확도** | 87.7%~91.4% | 같은 저자의 글을 높은 정확도로 식별 |
| **AA Top-5 정확도** | 79.8%~95.5% | 상위 5명 후보 안에 정답 저자가 포함 |
| **스타일 정확도** | 69.1%~81.9% | 저자의 스타일 모델과 실제 텍스트가 잘 매칭 |
| **인간 탐지율** | 99.2%~100% | 인간이 쓴 글이 거의 완벽하게 인간으로 탐지됨 |

→인간 작성 텍스트가 개인화되고 구별 가능한 스타일 패턴을 보인다는 것을 확인 이는 평가 프레임워크의 효과성을 정당화가능

#### 파인튜닝을 사용하지 않은 이유

파인튜닝(fine-tuning)이 종종 개인화된 생성을 달성하는 신뢰할 수 있는 방법이지만 본 연구에서는 의도적으로 배제

1. **데이터 부족**: 실제 사용자는 일반적으로 제한된 수의 글쓰기 샘플만 제공하며, 이는 효과적인 파인튜닝에 불충분
2. **비용 문제**: 각 사용자를 위한 별도의 파인튜닝된 모델을 유지하는 것은 비용이 많이 들고 대규모로는 비실용적
3. **지연 시간**: 여러 번의 왕복이 필요한 복잡한 프롬프팅 기법은 상당한 지연과 API 오버헤드를 유발

**대신 채택한 접근법**: 그래픽 인터페이스를 통해 사용자가 일반적으로 LLM과 상호작용하는 방식에 기반한 **더 실용적인 접근 방식**을 채택. API 호출을 한번 수행하여few-shot 프롬프팅에 초점을 맞추고 zero-shot 시나리오와 비교

### 실험 설정

#### 평가 대상 LLM

6개의 최첨단 모델을 평가:

| 제공사 | 모델 | 특징 |
| --- | --- | --- |
| **OpenAI** | GPT-4o, GPT-4o-mini | 범용 최고 성능 |
| **Google** | Gemini-2.0-Flash, Gemma-3-27B | 빠른 추론 속도 |
| **기타** | DeepSeek-V3 | 671B 파라미터 (37B 활성화) MoE |
| **Meta** | Llama-4-Maverick | `400B 파라미터 (17B 활성화) MoE` |

> MoE (Mixture of Experts): 전체 모델이 아닌 일부만 활성화하여 효율적으로 추론하는 구조
> 

**프롬프팅 설정**: 출력 재현성을 극대화하기 위해 **온도(temperature) 0**으로 설정 (가장 확률이 높은 토큰만 선택하는 탐욕적 디코딩)

#### 실험 조건

| 요소 | 설정 |
| --- | --- |
| **글쓰기 예시 샘플링** | 각 테스트 샘플마다 같은 저자의 훈련 세트에서 **무작위로 5개** 선택 |
| **테스트 샘플 요약** | GPT-4.1을 사용하여 각 테스트 샘플을 요약 (콘텐츠 통제) |
| **기준선(Baseline)** | Zero-shot (콘텐츠 요약만 제공, 예시 없음) |
| **Zero-shot 평가 모델** | 4개 모델만 사용 (DeepSeek, Llama 제외) |

#### 평가 모델 구축

| 평가 요소 | 사용 모델/도구 |
| --- | --- |
| **AA/AV 모델** | Longformer-base-4096, ModernBERT-base로 훈련 |
| **스타일 모델** | LIWC + WritePrint 특징 세트 기반 구축 |
| **AI 탐지** | GPTZero (사전 학습된 도구 사용) |

> LIWC (Linguistic Inquiry and Word Count): 심리언어학적 특징을 추출하는 도구. 감정, 인지적 과정, 사회적 관계 등을 분석
> 
> 
> **WritePrint**: 구문-어휘 패턴(문장 길이, 구두점 사용, 어휘 다양성 등)을 추출하는 스타일 분석 도구
>

### **결과**

#### 5.1 저자 귀속 정확도 (AA)

**평가 방법**: LLM이 생성한 텍스트를 저자 귀속 모델에 입력했을 때, 실제 저자가 **상위 5명 후보 안에 포함**되는 비율 측정 (Top-5 정확도)

#### 주요 발견

**Few-shot이 Zero-shot보다 일관되게 우수**

- 이전 샘플 제공이 LLM이 대상 저자와 스타일적으로 더 일치하는 텍스트를 생성하는 데 도움이 됨을 확인

**Zero-shot도 일부 성공**

- Zero-shot 생성도 무시할 수 없는 top-5 AA 정확도를 달성
- 이는 주로 개인의 글쓰기에서 나타나는 **주제/콘텐츠 중복**에 기인 (스타일보다는 내용이 저자 식별에 기여)

**데이터셋별 차이**

- **CCAT50 (뉴스)**: 일관되게 높은 정확도 → 언론인들이 별개의 주제를 다루기 때문
- **Blog, Reddit**: 변동성이 큼 → 비공식적이고 다양한 스타일

**모델별 차이**

- **Gemini-2.0-Flash와 Gemma-3-27B**: Reddit 및 블로그 같은 비공식 영역에서 상대적으로 우수한 성능
- 이유: 특수 문자(*, #, !) 등 비공식적 스타일 요소를 더 잘 포착

**저자별 분석**

- 모든 저자에 대해 zero-shot보다 few-shot에서 더 높은 정확도

#### 5.2 저자 검증 정확도 (AV)

**평가 방법**: LLM이 생성한 텍스트와 실제 저자의 텍스트를 저자 검증 모델에 입력했을 때, 같은 저자의 것으로 판단하는 비율 측정

#### 결과 요약

| 데이터셋 | 5-shot 범위 | 0-shot 범위 | 격차 |
| --- | --- | --- | --- |
| **CCAT50** (뉴스) | 94.68%~97.46% | 93.14%~94.18% | 소폭 차이 |
| **Enron** (이메일) | 95.65%~96.64% | 84.98%~85.95% | 약 10% |
| **Reddit** (포럼) | 49.97%~65.88% | 48.75%~56.66% | 중간 성능 |
| **Blog** (블로그) | 16.72%~21.25% | 8.15%~10.35% | 매우 낮음 |

#### 주요 발견

**모든 모델과 데이터셋에서 5-shot이 0-shot을 일관되게 능가**

**구조적 vs 비구조적 글쓰기**

- **공식적 글쓰기 (CCAT50, Enron)**: 90% 이상의 높은 정확도
    - 이유: 구조화되고 예측 가능한 형식, 일관된 어조
- **비공식적 글쓰기 (Reddit, Blog)**: 50% 이하의 낮은 정확도
    - 이유: 스타일적으로 다양하고 미묘한 개인적 뉘앙스

**해석**: LLM은 형식적 패턴과 구조는 잘 학습하지만, **개인의 미묘한 표현 방식이나 독특한 어투**는 재현하기 어려움

#### 5.3 스타일 모델링 및 정렬

**평가 방법**: 각 저자에 대해 언어적 특징의 **통계적 분포**를 기반으로 한 스타일 모델을 구축하고, 생성된 텍스트가 이 모델과 얼마나 가까운지 측정

#### 사용된 특징 세트

1. **LIWC (Linguistic Inquiry and Word Count)**
    - 심리언어학적 특징: 감정 표현, 인지적 과정(예: "생각하다", "알다"), 사회적 관계 표현
    - 예: 긍정/부정 감정 단어 사용 빈도, 1인칭/3인칭 대명사 사용 패턴
2. **WritePrints**
    - 구문-어휘 패턴: 평균 문장 길이, 구두점 사용, 어휘 다양성, 기능어 빈도
    - 예: 쉼표 사용 빈도, 접속사 선호도, 문장 복잡도
    
    **거리 측정: 마할라노비스 거리**
    
    - *마할라노비스 거리(Mahalanobis Distance)**란?
    - 두 데이터 포인트 간의 거리를 측정하되, **변수 간 상관관계와 분산을 고려**하는 통계적 거리
    - 유클리드 거리와 달리, 데이터의 분포 형태를 반영
    - **낮은 값** = 더 유사함 (저자의 스타일에 가까움)
    - **높은 값** = 덜 유사함 (저자의 스타일에서 멀어짐)

#### 주요 발견

**거리 순서 (낮음 → 높음)**

1. **원본 인간 샘플**: 가장 낮은 거리 (당연히 저자 자신의 스타일과 가장 일치)
2. **Few-shot 생성**: 중간 거리
3. **Zero-shot 생성**: 가장 높은 거리

**통계적 유의성**

- **Wilcoxon signed-rank test**(윌콕슨 부호 순위 검정)로 검증
    
    > 두 관련 표본의 차이를 검정하는 비모수 통계 방법. 여기서는 5-shot과 0-shot 생성의 스타일 거리 차이가 통계적으로 유의미한지 확인
    > 
- Few-shot 생성이 zero-shot보다 대상 스타일에 **통계적으로 유의미하게** 더 가까움

**인사이트**

- 예시 프롬프트 없이 LLM은 **일반적이고 평균적인 스타일**로 기본 설정됨
- 예시를 제공하면 개선되지만, 여전히 실제 저자의 스타일과는 상당한 간격 존재

**데이터셋별 변동**

- 저자 귀속 및 검증 성능 추세와 일치
- 공식적 글 > 비공식적 글 순으로 스타일 정렬 성능 우수
- 

#### 5.4 AI 생성 탐지

- AI가 쓴 글인지 탐지

**평가 도구**: **GPTZero**

> GPTZero는 AI가 생성한 텍스트를 탐지하는 상용 도구입니다. 텍스트의 "당혹도(perplexity)"와 "폭발성(burstiness)"을 분석하여 AI 생성 여부를 판단함
> 
> - **Perplexity (당혹도)**: AI는 예측 가능한 패턴으로 글을 쓰는 경향 (낮은 당혹도)
> - **Burstiness (폭발성)**: 사람은 문장 길이와 복잡도가 불규칙하지만, AI는 더 균일함

**평가 방법**: LLM이 생성한 텍스트를 GPTZero에 입력하여 **"사람이 쓴 글"로 분류되는 비율** 측정

#### 결과 요약 (5-shot 기준)

| 데이터셋 | GPT-4o | GPT-4o-mini | Gemini-2.0-Flash | Gemma-3-27B |
| --- | --- | --- | --- | --- |
| **CCAT50** | 0.08% | 0.00% | 20.49% | 31.36% |
| **Enron** | 16.86% | 13.76% | 54.25% | 43.63% |
| Reddit | 0.67% | 0.12% | 20.77% | 16.09% |
| **Blog** | 0.47% | 0.35% | 18.74% | `7.58%` |

#### 주요 발견

**대부분 LLM 생성물은 쉽게 탐지됨**

- 전체적으로 55% 미만, 대부분 20% 미만이 "사람이 쓴 글"로 분류
- 스타일을 모방하더라도 **AI 특유의 흔적**이 남음

**GPT 모델의 극단적 탐지율**

- GPT-4o, GPT-4o-mini: 거의 0%에 가까운 인간 탐지율
- **가설**: GPTZero가 **GPT 기반 모델의 출력을 탐지하도록 특별히 최적화**되었을 가능성
    - GPTZero 학습 데이터에 GPT 생성물이 많이 포함되었을 수 있음
    - 탐지기 편향(detector bias) 존재 가능성

**Google 모델의 상대적 우수성**

- Gemini-2.0-Flash, Gemma-3-27B: 더 높은 인간 탐지율
- 특히 **구조화된 데이터셋 (Enron, CCAT50)**에서 두드러짐
- 이유 추정:
    1. 다른 학습 데이터/방법론으로 인한 생성 패턴 차이
    2. GPTZero의 탐지 알고리즘이 이들 모델에 덜 최적화됨

**결과 해석 시 주의점**

- 탐지율 차이가 **진정한 스타일 유사성**을 반영한다기보다는 **탐지기의 모델별 편향**을 반영할 가능성
- 단일 탐지 도구에만 의존한 한계

**인사이트**

- LLM이 개인 스타일의 일부 특징을 모방할 수 있지만, 여전히 **"AI가 쓴 글"의 특성**을 강하게 유지
- 진정한 의미의 "사람처럼 쓰기"는 아직 달성되지 않음

#### 5.5 의미적 유사성 (Semantic Similarity)

**목적**: 생성된 텍스트가 원본 텍스트의 **내용(의미)**을 얼마나 잘 따르는지 확인

> 스타일 평가에 집중하느라 내용이 엉뚱하게 생성되지 않았는지 점검하는 검증 절차
> 

#### 평가 지표

1. **METEOR**
    - 기계 번역 평가를 위해 개발된 지표
    - 단어 일치뿐 아니라 동의어, 어간(stem) 등을 고려
2. **ROUGE-L**
    - 가장 긴 공통 부분 수열(Longest Common Subsequence) 기반
    - 텍스트 간 순서를 고려한 중복도 측정
3. **SBERT (Sentence-BERT)**
    - BERT 기반 문장 임베딩으로 의미적 유사성 측정
    - 0~1 사이 값 (1에 가까울수록 유사)

#### 결과

| 모델 | 설정 | METEOR | ROUGE-L | SBERT |
| --- | --- | --- | --- | --- |
| GPT-4o | 5-shot | 0.32 | 0.24 | **0.77** |
|  | 0-shot | 0.30 | 0.21 | 0.76 |
| GPT-4o-mini | 5-shot | 0.31 | 0.23 | **0.77** |
|  | 0-shot | 0.29 | 0.20 | 0.76 |
| Gemini-2.0-Flash | 5-shot | 0.32 | 0.26 | **0.78** |
|  | 0-shot | 0.31 | 0.21 | 0.75 |
| Gemma-3-27B | 5-shot | 0.30 | 0.23 | **0.75** |
|  | 0-shot | 0.29 | 0.19 | 0.74 |

#### 주요 발견

**높은 의미적 유사성**

- 모든 모델에서 SBERT ≥ 0.74 (상당히 높은 수준)
- LLM이 제공된 **콘텐츠 요약을 충실히 따름**

**일관된 패턴**

- 5-shot이 0-shot보다 세 가지 지표 모두에서 약간 더 높음
- 스타일 정렬이 향상되면서 **의미적 정확도도 함께 개선**

**해석**

- 스타일 충실도가 높아지면서 발생하는 부산물:
    - 단어 선택(word choice)이 원문과 유사해짐
    - 문장 구조(phrasing)가 원문과 비슷해짐
- 이러한 요소들이 의미적 유사성 점수를 높이는 데 기여

**실험 설계의 타당성 확인**

- LLM이 **지시(instruction)를 잘 따른다**는 것을 입증
- 스타일 평가에 집중하면서도 **내용 품질을 유지**했음을 확인

### 후속 연구: 프롬프팅 전략의 효과

실제 사용자는 무작위로 예시를 제공하기보다 **더 전략적인 선택**을 할 수 있습니다. 이에 따라 다양한 예시 선택 전략의 효과를 추가 검증했습니다.

#### 6.1 실험 조건

실험의 공정한 비교를 위해 **동일한 서브셋**에서 평가:

- CCAT50: 30명 저자 × 10개 샘플
- Enron: 30명 저자 × 10개 샘플
- Reddit: 50명 저자 × 10개 샘플
- Blog: 50명 저자 × 10개 샘플
- 평가 모델: GPT-4o, Gemini-2.0-Flash, Llama-4-Maverick (비용 효율성)

#### 테스트한 전략들

1. **콘텐츠 유사성 기반 선택 (+Sim ctrl)**
    - **방법**: **BERTopic**으로 저자의 모든 글을 주제별로 클러스터링
        
        > BERTopic: BERT 임베딩과 클러스터링을 결합한 토픽 모델링 기법. 문서를 의미적으로 유사한 그룹으로 자동 분류
        > 
    - 테스트 샘플과 **같은 클러스터(같은 주제)**에 속하는 5개 훈련 샘플 선택
    - **가설**: 같은 주제를 다룬 글을 예시로 주면 더 잘 모방할 것
    - **실제**: RAG(Retrieval-Augmented Generation)와 유사한 방식
2. **길이 정렬 (+Len ctrl)**
    - **방법**: 테스트 샘플과 **길이가 가장 유사한** 5개 훈련 샘플 선택
    - **가설**: 비슷한 길이의 글을 예시로 주면 스타일 일관성이 향상될 것
    - **근거**: 저자성(authorship)은 텍스트 길이에 따라 달라지는 특징이 있음
3. **스니펫 포함 (+Snippet)**
    - **방법**: 기본 5-shot 예시 + **원본 텍스트의 앞부분** 제공
    - **제공 부분**: 첫 50단어 또는 전체의 20% 중 짧은 것
    - **가설**: 실제 저자의 글 일부를 시작점으로 주면 자연스럽게 이어서 쓸 것
    - **평가**: 계속 작성한 부분만 평가 (제공된 스니펫 제외)
4. **예시 수량 변화**
    - **방법**: 2, 4, 6, 8, 10개의 예시로 테스트
    - **조건**: 각 세트는 이전 세트의 부분집합 (공정한 비교)
    - **가설**: 더 많은 예시 = 더 나은 스타일 학습

#### 6.2 주요 발견

#### 콘텐츠 유사성 선택 (+Sim ctrl) - 예상 외 부정적 효과

**결과**:

- 특히 **Enron, Reddit, Blog**에서 귀속 성능 **감소**
- Enron 예시: AA 69.33% → 36.00% (거의 절반으로 하락!)
- Blog 예시: AV 19.40% → 10.33%

**왜 이런 일이?**

- **긍정적 효과**: 주제 정렬(topical alignment) 향상
- **부정적 효과**: **스타일적 다양성(stylistic diversity) 감소**
    - 예시를 좁은 주제 클러스터로 제한
    - 저자의 다양한 스타일 표현을 보여주지 못함
    - 모델이 **저자 고유의 특성**보다는 **주제 특성**을 더 많이 학습

**통찰**:

- 같은 주제의 글이라도 **저자의 스타일은 다양**하게 나타남
- 무작위 샘플링이 오히려 **스타일의 폭넓은 스펙트럼**을 보여줌
- 내용 유사성 ≠ 스타일 유사성

---

#### 길이 정렬 (+Len ctrl) - 혼합된 결과

**결과**:

- **AA/AV**: 약간의 개선 (특히 CCAT50, Reddit)
    - CCAT50: AA 89.72% → 90.16%
    - Reddit: AV 68.07% → 70.07%
- **스타일 모델**: 정확도 **하락**
    - CCAT50: Style 60.11% → 57.89%
    - Enron: Style 72.11% → 61.22%

**왜 이런 패턴이?**

- **AA/AV 개선 이유**:
    - 텍스트 길이 자체가 **표면적 일관성** 신호
    - 길이가 비슷하면 문장 수, 단락 구조 등이 유사
    - AA/AV 모델이 이러한 **구조적 패턴**을 포착
- **스타일 모델 하락 이유**:
    - 길이 매칭이 **스타일 변동성(stylistic variation)**을 의도치 않게 제한
    - 저자는 짧은 글과 긴 글에서 **다른 스타일 특징**을 사용
    - 예: 짧은 글 - 간결한 표현, 긴 글 - 복잡한 문장 구조
    - 한 가지 길이만 보면 저자의 **스타일 레퍼토리** 일부만 학습

**통찰**:

- 저자 식별(AA/AV)과 스타일 충실도(Style)는 **다른 측면**을 평가
- 표면적 일관성이 항상 진정한 스타일 모방을 의미하지는 않음

---

#### 스니펫 포함 (+Snippet) - 가장 효과적

**결과**:

- **인간 탐지율에서 가장 강력한 증가**
    - CCAT50: 8.50% → 15.50% (약 2배)
    - Enron: 36.83% → 46.50%
    - Reddit: 10.90% → 22.10% (약 2배)

**왜 효과적인가?**

1. **진정성 있는 시작점**
    - 실제 저자의 글로 시작하면, 그 **톤(tone)**과 **리듬(rhythm)**을 자연스럽게 이어감
    - 저자 고유의 어투, 문체가 초반에 설정됨
2. **맥락의 연속성**
    - 전체 글의 흐름이 자연스러워짐
    - 이어쓰기(continuation)는 처음부터 쓰기보다 쉬움
3. **AI 특유의 시작 패턴 회피**
    - LLM이 처음부터 쓸 때 나타나는 전형적인 패턴(예: "In this essay...", "First of all..." 등) 회피
    - GPTZero 같은 탐지기가 포착하는 AI 특유의 시작 구조를 우회

**트레이드오프**:

- AA/AV 지표에서는 **혼합된 결과**
- 일부는 개선, 일부는 하락
- 이유: 인간다움과 저자 식별 가능성은 다소 다른 차원

**실용적 시사점**:

- 사용자가 초고를 조금 쓰고 LLM에게 이어쓰기를 시키는 방식이 효과적
- 완전히 LLM에게만 의존하기보다는 **협업적 글쓰기** 접근

---

#### 예시 수 증가 - 제한적 효과

**결과** (그림 5, 6 참조):

- 2개 → 4개 → 6개 → 8개 → 10개로 늘려도
- **네 가지 지표 모두에서 거의 변화 없음**
- 5개 이후부터는 **평탄화(plateau)** 현상

**왜 더 많은 예시가 도움이 안 되는가?**

1. **맥락 창(context window)의 한계**
    - LLM이 처리할 수 있는 토큰 수는 제한적
    - 예시가 많아지면 각 예시에 대한 **주의(attention)**가 분산
    - 초반 예시와 후반 예시에 대한 가중치가 달라질 수 있음
2. **스타일 학습의 포화**
    - 5개 정도면 저자의 주요 스타일 특징은 이미 포착
    - 추가 예시는 **새로운 정보**를 주지 못하고 중복
    - 오히려 **혼란(confusion)**을 줄 수도 있음 (예시 간 미세한 차이)
3. **In-context learning의 근본적 한계**
    - Few-shot 프롬프팅은 **명시적 학습**이 아님
    - 모델의 가중치가 업데이트되지 않음
    - 단순히 패턴 매칭에 의존하는데, 이는 **상한선**이 있음

**인사이트 :**

- **"More is better"가 항상 성립하지 않음**
- 5-shot이 **sweet spot** (비용 대비 효과)
- 10-shot 이상은 비용만 증가하고 성능 개선 없음

#### 핵심 정리

1. **예시 선택은 복잡한 문제**
    - 직관적으로 좋아 보이는 전략(주제 유사성)이 실패할 수 있음
    - 콘텐츠 유사성 ≠ 스타일 유사성
2. **단일 최적 전략은 없음**
    - 데이터셋/목적에 따라 효과적인 전략이 다름
    - Human 탐지율 ↑ 원하면 → +Snippet 또는 +Len ctrl
    - AA/AV 정확도 ↑ 원하면 → +Len ctrl
    - 균형잡힌 성능 원하면 → 기본 5-shot 무작위
3. **효과적인 개인화의 조건**
    - **도메인 민감성(Domain Sensitivity)**: 장르별로 다른 접근 필요
    - **의미 제어(Semantic Control)**: 내용과 스타일의 균형
    - **스타일 충실도(Style Fidelity)**: 표면적 유사성 vs 깊은 특징
    - **인간다움(Human-likeness)**: AI 탐지 회피
4. **실용적 권장사항**
    - **일반 사용자**: 5-shot 무작위 + 초고 일부 작성 후 이어쓰기
    - **고급 사용자**: 도메인별 전략 선택
    - **개발자**: 멀티 전략 앙상블 고려
