### 요약

Verbalized Sampling (VS)'은 LLM이 생성하는 응답의 다양성(semantic diversity)을 크게 높이는 새로운 프롬프트 엔지니어링 기법

이 기법은 모델에 단일 응답 대신 여러 응답과 각 응답의 확률을 함께 제시하도록 지시하여, LLM 내부의 다양한 가능성을 노출시켜줌

출력 결과물이 밋밋할 때 / 단어, 문장 반복이 많을 때 사용하기 좋음 단, 정확성을 요구하는 태스크에는 사용하지 않는게 좋다고 논문에서 언급됨 

프롬프트 사용 예시 : 

기존 : 곰에 대한 짧은 이야기를 써줘

VS : 곰에 대한 짧은 이야기를 5개 써주고, 각 이야기의 확률을 함께 알려줘

VS - Cot 응용 : 먼저 단계별로 생각한 뒤, 곰에 대한 짧은 이야기 5개와 각 농담의 확률을 알려줘


### 초록

post-training alignment는 LLM의 다양성을 감소시키는 경향이 있음 → mode collapse

mode collapse를 우회하기 위한 프롬프팅 전량 Verbalized Sampling(VS)를 소개

VS는 모델에 응답 집합에 대한 확률 분포를 언어화하도록 요청

- 커피에 대한 5개의 농담을 생성하고 확률을 제시하세요

→ 창작 글쓰기(시, 이야기, 농담), 대화 시뮬레이션, 개방형 질답, 합성 데이터 생성에서 정확성과 안정성을 손상시키지 않으면서 성능을 크게 향상시키는 결과를 보여줌 

### 서론

- RLHF나 데이터 편향 또는 샘플 선택 과정의 편향 같은 사후 학습 정렬 방법은 의도치 않은 mode callapse를 유발함

#### mode callapse란?

- 모델이 자신의 출력 중에서 좁은 범위로 응답하는 것
- 출력 다양성을 크게 감소시킴
- 창의적 글쓰기, 사회 시뮬레이션, 합성 데이터 생성 등 다양한 애플리케이션에서 LLMs의 효과를 제한함
    
    ⇒ 모델이 훈련 데이터의 전형적인 패턴(평균적인 표현)에 지나치게 맞춰져 있기 때문
    
    이걸 선호도 편향(preference biase)라고 부르고, 이는 단순히 RLHF나 데이터 문제 때문이 아니라 모델의 내적 표현 구조에서 비롯된다고 주장함
    

전향성 편향이 데이터 전반에 퍼져 있으니, 단순히 훈련 데이터 방식을 고치는 것으로는 부족함 → 프롬프팅으로 접근

#### Verbalized Sampling(VS)란?

- 샘플링의 지시를 언어로 표현하여 모델이 내부적으로 다양한 표현 공간을 탐색하도록 유도
- 응답 분포를 명시적으로 요청하고 해당 확률을 언어화하여 LLMs의 내재된 생성 다양성을 복원
    - 기존 : 이 질문에 답하시오
    - VS : 이 질문에 대해 확률과 함께 5가지 응답을 생성해줘

#### 효과

- 창의적 글쓰기에서 출력 다양성을 크게 향상됨
- 사회적 대화 시뮬레이션에서 VS가 훨씬 더 인간과 유사한 행등을 보이며, 일부 모델은 전용 미세 조정 모델과 동등한 성능을 보임
- 여러 유효한 답변이 있는 개방형 QA 작업에서 더 넓고 현실적인 응답 분포를 생성함
- 합성 데이터 생성에서 VS가 다운스트림 수학 작업 성능을 향상시킴 → 더 다양한 합성 데이터를 생성
- 또한 VS가 모델의 사실적 정확성과 안전성을 희생하지 않고 성능을 향상 시킴

### 전형성 편향(typicality bias)과 선호 데이터: 인지적 및 경험적 증거

#### **전형성 편향 가설(Typicality Bias Hypothesis)이란?**

- (인지심리학 연구에서) 전형적이거나 익숙한 것들을 선호하는 경향이 있다
- ex) 낯선 단어보다 자주 노출된 단어를 더 쉽게 회상하거나 더 선호하는 경향

→ 인간이 진실성보다 전형성에 더 높은 가치를 부여하는 경향이 있다. 

따라서 우리는, 인간이 일반적으로 전형성 편향을 가지고 있으며 이는 선호 데이터에도 반영이 되어있을 것이다(가정)

#### **전형성 편향과 보상 모델링(모델링 관점).**

- 전형성 편향 가설에 따라 선호 데이터가 이런 편향이 있다고 가정
- 즉, 선호 데이터를 학습하여 만들어진 보상 함수는 사람의 전형성 편향을 간접적으로 학습하게 된다
    
    → 전형성이 높은 결과는 보상 함수의 값이 더 높게 측정될 것 
    
- LLM이 사람의 선호 데이터로 fine-tunning 될 경우 생성의 다양성이 감소하고 평균적인 표현(= 더 전형적인 문장 구조나 단어 선택)이 늘어나는 현사잉 관찰됨 → Llama3나 GPT-4 같은 모델에서도 비슷하게 나타남
- 인간의 선호 데이터를 기반으로 RLHF를 수행할 때도, 보상 모델이 이러한 ‘전형성 편향’을 학습하여 결과적으로 평균적인 표현으로 수렴하는 경향을 보인다.우리는 이 경향이 LLM의**모드 붕괴(mode collapse)**현상과 밀접히 관련되어 있다고 주장

#### **HTBCMC(Hybrid Token-Based Contrastive Monte Carlo) 기법 제안**

- 전형성 편향을 완화하기 HTBCMC를 제안
→ 저자들은 인간의 문장 생성 방식과 유사한 확률적 다양성을 회복할 수 있는 방법
- π∗(y∣x)=πref(y∣x)exp(β1r(x,y))/Z(x), 여기서 Z(x)Z(x)Z(x)는 정규화 상수다. 이 형태는 보상에 따라 확률 분포를 “샤프닝(sharpening)”하는 효과를 가지며, 높은 보상을 받는 출력을 더 선택적으로 강화한다.

#### Verbalized Sampling의 핵심 아이디어

- HTBCMC는 토큰 단위로 위 확률 분포를 적용하여, 문장 전체가 아니라 각 토큰마다 보상 기반 샘플링을 수행
    
    → 전형적인(평균적인) 문장 구조만 계속 선택되는 것을 방지하고,토큰 수준에서 다양성을 회복시켜 언어적 표현 다양성(verbal diversity)을 확보하는 것이 핵심이다.
    
- 단순히 temperature scaling)로 랜덤성을 주는 것과 달리, 보상 함수의 형태에 따라 의미 있는 다양성(semantic diversity)을 유도한다.
- 결과적으로, HTBCMC를 이용하면 LLM이전형성 편향을 억제하면서 RLHF 과정에서 사라진 창의성과 표현의 폭을 복원할 수 있다.

### 방법 - 언어와 행동(Verbal and Action Sampling)

#### CDM(Contrastive Decoding Module)

- 대부분의 LLM은 특정 프롬프트 x에 대해 샘플링된 토큰을 기반으로 단어 확률 분포를 계산함
    - 높은 확률을 가진 후보들이 과도하게 집중되면, 평균적인 문장만 생성하게됨 → 모드 붕괴
- CDM은 대조적 학습의 개념을 차용
    - 서로 다른 후보 토큰 간의 의미적 유사도와 보상 함수를 함께 고려
    - 가장 전형적인 표현보다는 의미적으로 유효하면서도 다른 표현을 선택하도록 유도

#### CDM 단계

1. 샘플링 : 후보 토큰 집합을 생성
2. 대조적 선택 단계 : 각 후보의 보상과 의미적 차이를 계산해서, 전형성(typicality)이 낮으면서도 의미적 일관성이 높은 후보를 선택한다

→ 이 과정을 통해 CDM은 모델이 비슷한 의미를 유지하면서도 다른 방식의 표현을 학습하도록 유도

#### Verbalized Sampling(VS)

- Verbalized Sampling(VS)은 CDM에서 생성된 다수의 후보를 ‘언어적 형태로 표현(verbalize)’한 뒤,이를 다시 모델의 입력으로 활용하는 메타-샘플링(meta-sampling) 기법

핵심 아이디어 :

- 모델이 단일 확률 분포에서 한 번의 샘플링으로 결정되는 것이 아니라, **자신이 생성한 후보들을 ‘언어적으로 설명하고 평가’하도록 유도**한다.
- 이렇게 함으로써 모델이 ‘생성 행위 자체를 언어화’하며, 이는 인간의 자기 설명적 사고(self-explanatory reasoning)와 유사한 효과를 낸다.

#### VS의 효과 및 직관

- Verbalized Sampling은 단순한 무작위성 증가가 아니라, **‘생성 과정에 대한 메타 인식(meta-awareness)’을 추가함으로써** 모델의 표현 다양성을 확보한다.
- 이 접근은 기존의 top-k, temperature, nucleus sampling과 달리,언어적 후보들 간의 **의미적 차이와 설명 능력**을 동시에 고려한다는 점에서 차별화된다.
- 실험적으로, VS를 적용한 모델은 RLHF로 학습된 LLM에서 흔히 나타나는 모드 붕괴를 크게 완화했다.예컨대 GPT-4, Claude-4, LLaMA-3 등의 모델에서 VS를 적용했을 때, 표현 다양성 점수(diversity score)가 최대 **37%** 향상되었다.

### 창의적 글쓰기

#### 실험

1. 시 완성
2. 스토리 생성
3. 농담 생성 

총 3가지의 과제를 중심으로 실험

#### 벤치마크

- **(1) 시 생성:**
    
    Lu et al. (2025b)의 텍스트 연속 생성(text continuation) 설정을 따른다.
    
- **(2) 스토리 생성:**
    
    Turgeman et al. (2025)의 “Books and Movies” 데이터셋을 활용하며,
    
    총 100개의 프롬프트를 무작위로 선택해 사용한다.
    
- **(3) 농담 생성:**
    
    Reddit에서 수집된 DadJokes 데이터셋(Reddy, 2012)을 기반으로 한다.
    
    우리는 “setup(도입부)”이 주어졌을 때 “punchline(결말)”을 생성하도록 모델을 학습시킨다.
    
    학습 샘플의 수를 균형 있게 유지하기 위해, 각 데이터셋당 100개의 프롬프트를 사용한다.
    
    각 프롬프트에 대해 5개의 샘플을 생성하여 결과를 평가한다.
    

#### **평가 (Evaluation)**

모든 모델은 다양성(diversity)과 **품질(quality)** 두 가지 측면에서 평가

1. **다양성(Diversity):**
    - 문장 수준의 의미 다양성을 평가한다.
    - 구체적으로는 Sentence-BERT(Reimers & Gurevych, 2019)를 활용하여 생성된 문장 간 임베딩의 평균 코사인 거리(semantic diversity score)를 계산한다.
2. **품질(Quality):**
    - 문법적, 문체적 자연스러움과 인간 평가자의 주관적 선호를 함께 고려한다.
    - BLEU(Papineni et al., 2002), ROUGE(Lin, 2004), GPTScore(He et al., 2023a) 등의 자동 평가 지표를 사용하고, 사람 평가(HumanEval)에서는 HumorBench(Narad et al., 2025a)의 평가 루브릭을 따랐다.

#### **다양성 점수 (Diversity Score)**

- Verbalized Sampling(VS) 기반 변형들(VS-CoT, VS-Multi)은 기존 대비 일관된 다양성 향상을 보였으며,특히 언어적 다양성과 의미적 다양성의 균형 측면에서 가장 우수한 결과를 나타냈다.

### 대화 시뮬레이션

- VS가 인간 수준의 대화 다양성과 설득력에서 어떤 성능을 보이는지를 **PersuasionForGood** (Wang et al., 2019) 데이터셋을 통해 평가했다
    - 페르소나를 가진 참가자들이 상대방에게 자선 기부를 설득하는 대화로 구성된 데이터 세트임
    - LLM이 상태방의 반응을 예측하고 설득하는 능력을 평가하기에 적합
    - 실제 인간 간 대화의 복잡한 상호작용 패턴을 모사할 수 있는지를 검증하는데 유용

#### 실험 설정

- 설득 과제를 중심으로 설계
- LLM이 “기부를 설득하는 발화”를 생성하고, GPT-4-turbo가 “기부를 고려하는 사람”으로 응답하는 식
- 3가지 모델을 비교함
    - **기준 모델:** Llama3-8B 기반 표준 대화 모델
    - **VS 적용 모델:** Verbalized Sampling이 적용된 버전
    - **Persona-aware 모델:** 페르소나 정보를 명시적으로 포함한 버전

#### 평가 방법

1. **설득 성공률 (Persuasion Success Rate):** 상대방이 기부 의사를 명확히 밝힌 비율을 측정.
2. **인간 유사도 (Human-likeness):** 사람 평가자가 대화의 자연스러움, 감정 표현, 반응의 적절성을 평가.
3. **의미적 다양성 (Semantic Diversity):** Sentence-BERT 임베딩을 활용하여 대화 내 발화들의 의미적 분산도를 측정.
4. **가독성 (Readability):** Flesch–Kincaid Grade Level(Flesch, 1948) 지표를 사용해 텍스트 난이도를 측정.

#### 결과

1. **설득 성공률 향상:** VS 모델은 기준 모델보다 평균 **+9.4%** 높은 설득 성공률을 기록했다. 이는 VS가 보다 유연한 문체와 표현을 통해 상대방의 반응에 맞춰 언어를 조정할 수 있음을 시사한다.
2. **인간 유사도 향상:** 사람 평가자들은 VS 모델의 발화가 “덜 기계적이고 더 인간적”이라고 평가했다. 특히 감정적 표현과 문맥 적응성이 개선되었다.
3.  **다양성 증가:** VS 모델은 동일한 주제에 대해서도 더 다양한 어휘와 문장 구조를 사용했다. Sentence-BERT 기반 다양성 점수는 기준 모델 대비 약 **+21%** 향상되었다.
4.  **가독성 유지:** 흥미롭게도, 다양성이 높아졌음에도 불구하고 가독성 점수는 오히려 약간 개선되었다(평균 10.8 → 9.9 수준).

### 추론 시뮬레이션 (Reasoning Simulation)

Verbalized Sampling(VS)의 최종 응용 분야로, 우리는 VS가 **추론(reasoning)** 및 **안전성(safety)** 관련 과제에서 어떻게 작동하는지를 탐구

#### 실험 설정

1. **StrategyQA (Geva et al., 2021)** — 다중 단계의 일반 상식 추론(multi-hop commonsense reasoning) 문제로 구성되어 있다. 모델은 주어진 질문에 대해 "예/아니오(Yes/No)"로 답해야 하며, 그 과정에서 중간 추론 단계를 명시적으로 표현해야 한다.
2. **TruthfulQA (Lin et al., 2022)** — 모델이 잘못된 상식이나 허위 정보를 따라가지 않고, 사실적이고 신뢰성 있는 답변을 생성할 수 있는지를 평가한다.

#### 평가 지표

1. **정확도(Accuracy):** 모델이 올바른 최종 답변을 낸 비율.
2. **추론 일관성(Coherence):** 모델이 생성한 중간 추론 단계가 논리적으로 일관된 정도를 인간 평가자가 채점.
3. **안전성(Safety):** TruthfulQA의 자동화된 안전성 평가 루브릭에 따라, 사실적 오류(false factuality) 및 불필요한 허위 진술의 비율을 계산.

#### 결과

VS를 적용한 모델은 모든 항목에서 기준 모델보다 더 우수한 성능을 보였다.

1. **정확도 향상:** LLaMA3-8B 기준 모델 대비 VS-LLaMA3-8B 모델은 StrategyQA에서 **+5.8%**, TruthfulQA에서 **+4.2%** 높은 정확도를 기록했다.
    - 이는 VS가 단순한 샘플링 기법이 아니라, 추론 과정에서 불확실성을 탐색(exploration)하는 효과를 주기 때문으로 해석된다.
2. **추론 일관성 개선:** 인간 평가 기준에서 VS 모델은 중간 추론의 논리적 흐름과 근거 제시의 명확성에서 더 높은 점수를 얻었다.
    - 특히 Chain-of-Thought(CoT) 형태의 추론을 수행할 때, VS-CoT 변형이 가장 높은 일관성을 보였다.
3. **안전성 강화**: ****TruthfulQA 기준에서 VS 모델은 사실적 오류(falsehood)를 평균 **17% 감소**시켰다.
    - 이는 VS가 단순히 표현을 다양화할 뿐 아니라, 모델의 확신(confidence)을 적절히 분산시켜 ‘너무 확신에 찬 오류(overconfident mistake)’를 줄이는 역할을 한다는 점을 시사한다.

#### 분석

VS는 다양한 사고 경로를 탐색하면서도, 논리적 일관성이 높은 경로를 선택하도록 작동

VS는 다음과 같은 특성을 보였다

- 동일한 문제에 대해 여러 표현으로 접근하면서
- 그중 논리적으로 타당한 해석을 스스로 선택할 확률이 높아졌다

→ VS가 단순히 랜덤성을 높이는게 아니라 모델의 사고 공간을 넓히는 방법!

### Synthetic Data Generation (합성 데이터 생성)

- **VS가 생성 데이터의 다양성을 어떻게 개선하고, 결과적으로 모델 성능을 높이는지**를 실험적으로 검증

#### 실험 설정

- **기본 모델:** GPT-4.1 및 Gemini 1.5 Pro.
- 각 모델은 수학, 과학, 역사, 사회 등 다양한 분야의 문제를 생성하도록 프롬프트를 부여받았다.
- 각 실험에서 1,000개의 질문을 생성하며, VS와 기존 방법(RLHF, CoT 샘플링 등)을 비교했다.
- 일부 실험에서는 Qwen2-3B 모델을 사용해 실제 합성 데이터로 미세조정(fine-tuning)도 수행했다

#### 비교 대상

- **Standard Sampling** – 일반적인 top-p + temperature 방식.
- **Contrastive Sampling (CS)** – 대조적 학습 기반 샘플링.
- **Verbalized Sampling (VS)** – 제안된 방법.

#### 벤치마크 및 평가

결과적으로, VS로 생성된 합성 데이터로 학습된 모델은 **전 과제에서 일관되게 높은 성능**을 보였다.

- VS는 프리트레이닝 분포(pretraining distribution)와의 일치도를 유지하면서도
    
    답변 다양성(coverage)을 확보하는 데 성공했다.
    
- 특히, **precision-recall 균형이 1.0 근처**로 유지되며 안정적인 데이터 품질을 보였다.

### 한계점

- **계산 비용 증가 (Computational Overhead):**
    - VS는 여러 후보를 생성하고 이를 언어적으로 평가하는 과정을 포함하기 때문에, 기존 샘플링보다 **약 1.8배~2.5배의 계산 비용**이 필요하다.
    - 대규모 모델(>70B parameter)에서는 이 비용이 상당히 커질 수 있다.
- **Verbalization 품질의 의존성 (Dependence on Verbal Expression Quality):**
    - VS의 효과는 모델이 생성하는 언어적 설명의 품질에 따라 달라진다.
    - 모델이 불완전한 자기 설명(self-rationalization)을 할 경우, 오히려 오류가 증폭되거나 일관성이 떨어질 수 있다.
- **적용 범위의 제약 (Limited Scope):**
    - 본 연구는 주로 영어 기반의 일반 언어 모델(LLM)에 대해 실험되었다.
    - 다국어 모델이나 도메인 특화 모델(법률, 의학 등)에 대해 VS가 동일한 효과를 낼 수 있는지는 향후 검증이 필요하다.
- **정책 일관성 문제 (Policy Consistency):**
    - VS는 다양성을 강화하지만, 그 과정에서 동일한 프롬프트에 대해 응답의 스타일이나 결론이 달라질 가능성도 있다.
    - 즉, 일관성(consistency)과 **다양성(diversity)** 간의 균형을 세밀히 조정할 필요가 있다.
