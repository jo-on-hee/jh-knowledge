## 초록

#### 핵심 문제 제기

- 최신 대규모 추론 모델(LRM)은 ‘사고 과정’을 생성하여(CoT 기법 등) 성능을 높였지만, 그 능력과 한계는 잘 알려져 있지 않음
- 기존 평가는 최종 정답 정확도에만 초점을 맞추고 데이터 오염 문제가 있어 추론 과정 자체를 분석하기 어렵다

#### 연구 방법

- 복잡도를 정밀하게 조절할 수 있는 제어 가능한 퍼즐 환경을 사용해 LRM의 최종 답변뿐만 아니라 내부 사고 과정까지 체계적으로 분석

#### 주요 발견

1. LRM은 특정 복잡도를 넘어서면 정확도가 완전히 붕괴되는 현상
2. 복잡도가 높아질수록 추론에 더 많은 노력을 기울이다가, 특정 지점을 넘으면 오히려 추론 노력을 줄이는 한계를 보임 
3. 일반 LLM과 비교 시 낮은 복잡도에서는 일반 LLM이 중간 복잡도에서는 LRM이 우세하며 높은 복잡도에서는 둘 다 실패하는 세 가지 영역이 존재함

## 1. 서론

> 🖼️ [이미지 placeholder: 9.53.12.png]

- x축인 복잡도(디스크 수)가 증가함에 따라 사고하는 모델(+thinking)과 사고하지 않는 모델(claude 3.7)의 정확도 및 응답 길이(토큰 수) 변화를 보여줌
- 낮은 복잡도에서는 비사고 모델이 효율적, 복잡도가 증가하면 사고 모델이 우세하다가 결국 둘 다 성능이 붕괴됨

#### 서론의 핵심 질문

LRM이 진정한 의미의 추론을 하는 것인지, 아니면 정교한 패턴 매칭에 의존하는 것인지 근본적인 질문을 던짐

현재 멘치마크 평가 방식의 한계를 지적하며 통제된 실험 환경의 필요성을 다시 한번 강조함

#### 퍼즐 환경의 장점

- 문제의 복잡성을 세밀하게 조절 가능
- 기존 벤치마크와 달리 데이터 오염 문제가 없음
- 외부 지식 없이 주어진 규칙만으로 문제를 풀어야함
→ 알고리즘 추론 강조
- 풀이 과정의 모든 단계를 정확히 검증 가능

#### 주요 발견

- LRM은 특정 복잡도 이상헤서 성능이 0으로 붕괴되어 일반화에 실패함
- 복잡도가 임계점에 가까워지면 오히려 추론 노력을 줄이는 스케일링 한계를 보임
- 사고 과정을 분석한 결과, 간단한 문제에서는 정답을 찾고도 불필요한 탐색을 계속하는 과잉사고(overthinking) 현상이 발생함

⇒ 통제된 실험 환경에서 설계하고, LRM의 스케일링 한계를 밝혀냈으며 사고 과정 내부를 정량적으로 분석하여 자기 교정 매커니즘에 대한 통찰을 제공했다는 점에서 이 연구가 중요함

## 2. 관련 연구 언어 모델의 추론

- 초기 LLM의 낮은 추론 성능을 개선하기 위해 CoT(Chain of Thought) 기법이 도입되었고 이후 강화 학습을 통해 추론 능력을 특화시킨 LRM이 등장
- LRM이 사고 과정과 최종 답변이 불일치하는 현상이나, 정답을 찾고도 계속 연산을 수행하는 과잉 사고(overthinking)과 같은 비효율성을 보인다는 연구들이 있었음
- 기존 연구들이 주로 수학 벤치마크를 사용한 것과 달리 이 연구는 제어 가능한 퍼즐 환경(어려운 수학 사고 문제 정도?..)을 도입하여 문제 복잡도를 정밀하게 조작하고 추론 과정 자체를 깊이 있게 분석함으로써 LRM의 근복적인 능력과 한계를 더 명확히 규명하고자 함

#### 기존 수학 벤치마크의 문제점

- **MATH-500**: 사고/비-사고 모델 간 성능 차이가 거의 없음
- **AIME24, AIME25**: 사고 모델의 성능이 비-사고 모델보다 월등히 높게 나타남

⇒ 이런 일관성 없는 결과는 모델의 추론 능력이 향상되었기 때문인지 아니면 최신 벤치마크 데이터에 덜 오염되었기 때문인지 명확하지 않음

오히려 인간은 **AIME25를 더 쉽게 풀었는데 모델은 더 어려워하는 현상 발생 → 데이터 오염 가능성 시사** 

## 3 수학 및 퍼즐 환경

#### 3.1 제어 가능한 퍼즐 환경이란?

- **네 가지 퍼즐(문제)**:
    1. **하노이의 탑 (Tower of Hanoi)**: 원반을 규칙에 맞게 다른 기둥으로 옮기는 문제(**원반 수**로 복잡도를 조절)
    2. **체커 점핑 (Checkers Jumping)**: 다른 색깔의 체커 위치를 서로 바꾸는 문제(**체커 수**로 복잡도를 조절)
    3. **강 건너기 (River Crossing)**: 제약 조건 하에 사람들을 보트로 강 건너편으로 옮기는 문제(**사람(에이전트/고객 쌍) 수**로 복잡도를 조절)
    4. **블록 월드 (Blocks World)**: 흩어진 블록을 목표 형태로 다시 쌓는 문제(**블록 수**로 복잡도를 조절)
- 퍼즐(문제)들은 모두 **구성 요소의 수(N)를 조절**함으로써 문제의 복잡성을 체계적으로 증가시킬 수 있음

## 4. 실험 및 결과

#### **4.1실험 설정**:

주로 Claude 3.7 Sonnet과 DeepSeek 모델의 사고/비-사고 버전을 비교, 각 문제당 25개의 샘플을 생성해 평균 성능을 측정

- **실험 결과 (그림 4)**: 모든 퍼즐에서 복잡도가 증가함에 따라 사고/비-사고 모델의 정확도가 어떻게 변하는지 나타냄

#### 4.2 복잡성은 추론에 어떤 영향을 미치는가?

**4.2.1 세 가지 복잡도 영역(Three Regimes of Complexity):**

1. **낮은 복잡도 영역**: 비-사고 모델이 더 효율적이거나 성능이 비슷
2. **중간 복잡도 영역**: 사고 모델이 명백한 우위
3. **높은 복잡도 영역**: 두 종류의 모델 모두 성능이 **0으로 붕괴**
- 이 결과는 수학 벤치마크에서 보이지 않았던, 복잡도에 따른 뚜렷한 패턴

> 🖼️ [이미지 placeholder: 10.07.00.png]

- **Pass@k 성능 분석 (그림 5)**: 동일한 계산 비용(토큰 예산)을 주었을 때의 성공률을 비교
    - 낮은 복잡도에서는 비-사고 모델이, 중간 복잡도에서는 사고 모델이 우세하며, 높은 복잡도에서는 둘 다 실패하는 **세 영역을 명확히 확인**

#### 추론 모델의 붕괴

> 🖼️ [이미지 placeholder: 10.07.29.png]

**추론 모델의 붕괴 (그림 6)**:

- **정확도 붕괴**: 모든 LRM은 문제 복잡도가 특정 임계점을 넘어서면 정확도가 0으로 급격히 떨어지는 **'붕괴(collapse)' 현상 발생**
- **사고 토큰의 역설**: 모델이 이 붕괴 지점에 가까워질수록 문제 해결을 위해 더 노력하는 것이 아니라 오히려 **사고에 사용하는 토큰 수를 줄임
→**모델의 추론 능력에 근본적인 **스케일링 한계**가 있음을 의미

#### 4.3 추론 모델의 생각 속에서는 무슨 일이 일어나는가?

> 🖼️ [이미지 placeholder: 10.07.55.png]

**사고 과정 분석 (그림 7a)**: 사고 과정 내에서 정답과 오답이 나타나는 상대적 위치 분석

- **낮은 복잡도**: 정답을 **초반에 찾고도** 불필요한 오답 탐색(과잉 사고)
- **중간 복잡도**: 여러 오답을 탐색한 후 **후반부에** 정답에 도달
- **높은 복잡도**: 사고 과정 내에서 **정답을 전혀 생성하지 못함**

**사고 과정 내 정확도 변화 (그림 7b)**: '하노이의 탑' 예시에서, 쉬운 문제(N=1-3)는 사고가 진행될수록 정확도가 오히려 떨어지는 경향을 보이지만(과잉 사고의 증거), 중간 난이도 문제(N=4-7)는 사고가 진행될수록 정확도가 점차 높아짐 어려운 문제(N>=8)에서는 계속해서 정확도가 0임

#### 4.4 미해결 문제 : 추론 모델의 난해한 행동

> 🖼️ [이미지 placeholder: 10.08.37.png]

- **알고리즘을 알려줘도 실패 (그림 8a, 8b)**:
    - '하노이의 탑' 퍼즐의 **해결 알고리즘을 프롬프트에 명시적으로 제공**했음에도 불구하고, 모델의 성능은 전혀 향상되지 않았으며 이전과 동일한 복잡도에서 성능이 붕괴
    
    → LRM이 해결책을 '찾는 것'뿐만 아니라, 주어진 논리적 단계를 순서대로 '실행하는' 능력 자체에 근본적인 한계가 있음을 시사
    
- **퍼즐에 따른 일관성 없는 성능 (그림 8c, 8d)**:
    - Claude 3.7 모델은 '하노이의 탑'에서는 100번에 가까운 움직임을 오류 없이 수행하기도 하지만, '강 건너기' 퍼즐에서는 단 4~5번의 움직임 만에 오류 발생
    
    → '강 건너기'와 같이 웹상에 데이터가 드문 유형의 문제에 대해 모델이 제대로 학습(혹은 암기)하지 못했기 때문일 수 있음
    

## 5. 결론

- **결론**:
    - 현재의 LRM은 정교한 메커니즘에도 불구하고 특정 복잡도를 넘어서면 **일반화에 실패**하고 성능이 붕괴되는 근본적인 한계가 있음
    - 복잡도에 따라 세 가지 뚜렷한 성능 영역이 존재하며, 붕괴 직전 추론 노력을 줄이는 **스케일링 한계가 존재**
    - '과잉 사고' 현상이나, 명시적인 알고리즘을 실행하지 못하는 등의 한계는 현재 접근법이 일반화된 추론 능력 확보에 있어 근본적인 장벽에 부딪혔을 가능성
- **연구의 한계점**:
    1. **제한된 작업 범위**: 사용된 퍼즐 환경이 실제 세계의 다양하고 지식 집약적인 추론 문제를 모두 대표하지는 못함
    2. **블랙박스 분석**: 대부분의 최신 LRM을 API를 통해 분석했기 때문에, 모델 내부 구조를 직접 들여다보는 데 한계
    3. **검증 방식의 한계**: 퍼즐처럼 명확한 정답이 없는, 덜 구조화된 문제 영역에는 본 연구의 분석 방법을 그대로 적용하기 어려울 수 있음
